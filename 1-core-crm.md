## üö® ERROR PREVENTION & MITIGATION

| Rule | Action / Instruction |
| :--- | :--- |
| **The 3-WAY Rule** | **Critical Execution Verification** (Extension of Double Check):<br>1. **Say It:** Verbalize the exact command/action.<br>2. **Point It:** Point to the code/screen element.<br>3. **Do It:** Execute the action only after monitor confirmation. |
| **Double Check** | **Apply the Two-Person Rule** (Four-eye principle) for any irreversible or production-critical action (e.g., Purge, Delete, Config Change). |
| **The 8-Hour Rule** | **Fatigue Mitigation:** After **8 hours** of continuous critical incident work, **demand a mandatory break** or a full handoff to a fresh team. Cognitive functions degrade rapidly after this point. |
| **Role Clarity** | **Designated Roles:** In any major incident, explicitly name: **Incident Commander** (IC: owns plan/FOR-DEC), **Communicator** (Comms: external/status), and **Operator** (Ops: executes commands). **No one person handles all three.** |
| **Documentation Mandate** | **"Write It Down Now."** Live-document critical actions, commands, and outcomes in the incident journal **as they happen**. Don't rely on post-mortem memory. |
